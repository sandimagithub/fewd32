Other Notes

  • The hard part of coding is understanding the problem and figuring out how to solve it with the tools and code available
  • Always start with pseudo code...think about what you're trying to do and break it down into steps.
  • After you've done that, you just have to figure out how to express those steps in code :)

Decision Structures

  • AND: if all the conditions are true, the expression is true. Expressed like '&&'
  • OR: if one of the conditions is true, the expression is true. Expressed lik '||'
  • NOT: negation. if condition is true, the expression is false. If the expression is false, the expression is true. Expressed like '!='
  • IF / ELSE IF / ELSE: Evaluates condition branches until one of them is True and runs the body of the true condition branch. You can nest if / else conditions. KEEP TRACK OF YOUR BRANCHES!!!

Week 2

Intro

  • HTML are the "nouns" of a website. They give structure to your pages.
  • CSS are the "adjectives" that give your site more descritpion.
  • JavaScript are the "verbs" that brings your website to life and makes them more interactive.
  • A Common convention is to put community links first, then your links. All of these are placed in the head.

Structure

    Client-Server Architecture

  • Cloud: Serves up content remotely to all other client devices 9smart phones, laptops, etc).
  • Everything gets transfered in request response cycles: Your device asks the server for content, and the server responds back with your material. Your device then builds a local representation of what you asked for.
  • Ever wonder how you get thumbnail previews? Fetch thumbnails help you see content to reduce the number of requests.

    • The Backend

  • Dev-speak for all the stuff the user doesn't see. Its all about querying and organizing data, then sending that data to the front-end for everyone's viewing pleasure.

The DOM

  • Internally built tree-like representation of your HTML files in the browser
  • Lives in your browser's memory.
  • You can manipulate the DOM, mostly with JavaScript.
  • Use the Browser Developer Tools to help manipulate the DOM (hotkey: Ctrl + Opt + I)

HTML

    Elements

  • Tags or "pieces" of HTML that have their own properites and attributes(e.g. div, video, audio, p, etc)
  • Used to lay out information visually in the browser

    • The Box Model

  • Structure of HTML elements
  • Margin, Border, Padding, Width, Height, Content
  • Can manipulate each of these with code (mostly with CSS and JavaScript).

CSS

    CSS Selectors -- ID and Class

  • IDs and Classes are "hooks" for styling and manipulating HTML elements with CSS and JavaScript.
  • An ID is a unique hook...there is only one of each ID (e.g.one dog named sparky).
  • a Class is a common hook...there's a group of these (e.g. dogs).
  • Make your ID and Class names MEANINGFUL to you or anyone else working with your code so it makes sense to them.
  • Using IDs and Classes is all about matching the right "adjectives" and "verbs" with the right "nouns".
  • You can target HTML elements (div, p, span, etc) to style with CSS or manipulate with JavaScript

    • Color

  • Use can use keywords like "gray" or "teal" to add color.
  • You can also use hexadecimal values (e.g. #33F203 is bright green). Check this out: Hex Color Picker
  • The last way to use color is with RGBA values (e.g. rgba(0,0,0,1) = black). RGB are the amount of Red, Green, and Blue in the color, and A is alpha which is the transparency.
    Look at this: RGBA picker

    • Display Properties -- Inline vs Block

  • Different HTML elements start with different CSS properties.
  • Inline is "in-text, read left to right". Elements with in-line only take up the space they need.
  • Elements with the Block property wrap to the next line by forcing a break before and after the element

    • Display Properites -- Static vs Absolute

  • Static means the browser "makes room" for elements wtih this property in flow. This is the default for most elements.
  • The absolute property means the browser doesn't make room for the element, so other elements aren't aware of it. Its "ripped" out of the DOM.

Patterns

  • Common Layouts
  • Nav Headers are usually placed at the top to help users navigate your site, and have the absolute property.
  • Hero Images have position absolut, background-image in a div to display large images.
  • These are not set rules, just common arrangements to help you lay out sites quickly using components and sections.

Bootstrap

  • Start with 2 divs: 1) "container" to put all the elements affected by bootstrap in, and 2) "row" to create a new section in the container.
  • Bootstrap has a grid broken into 12 columns. Using rows and columns, you can position HTML anywhere on the page.
  • Choose how many columns you want each element or section to take up.
  • You can also set default behavior for different device sizes with designations (xs, md, lg, xl).
  • Using different device sizes, you can control the behavior of your website for each device size.

Week 3

Intro to JavaScript

Goal: control flow of data to enhance user experience by manipulating interface

High vs Low level languages:

  • high = scripting languages like JavaScript
  • low = machine level code like binary (has no understanding of human level ideas like "Lady Gaga")
  • Information from high level gets translated to low level so that computers do what we want
  • Fun fact: you can use javascript to interface wtih stuff on the Internet of Things!

    • Functions

  • Structure = keyword 'function' 'function name' (parameters){ body };
  • function = reserved word in JavaScript
  • function name = whatever you want the name of your function to be
  • parameters = variables you want to make room for in the function body
  • body = code that tells your function what to do

    • JQuery

  • Javascript library that makes it easy to query the DOM to manipulate HTML & CSS

    • CPU, Memory, and Storage

  • CPU = Central Processing Unit. Executes all the commands you give it from short term memory.
  • Memory = main memory (short term, volatile). When the computer shuts off, this gets blown away.
  • Storage = permanently store data(HTML/CSS/JS, music and text files, etc.)
  • Everything typically get referenced from Main Memory

    • Variables

  • Allow you to represent and work with data by declaring a variable name and assign a value.
  • Variables get stored in main memory
  • Declared using 'var' keyword (e.g. var name = 'Andy')

    • Data Types (Primitives)

  • Strings = anything in double or single quotes (e.g. 'Obama')

    • Note: Literals are anything in quotes and cannot be changed

  • Number = just what is sounds like...numerical values (e.g. 42)
  • Boolean = values referencing validity: True and False
  • Array = a list of values by index in brackets (e.g. [True, 'Barack', 42] )
  • Object = a list of values by key-value paris in curly braces (e.g. { name: 'Andy', age: 21} )
  • Undefined and Null = empty or undeclared values

    • Decisions & Logic

  • made of premises and conditions
  • branch statements: If, Else, Else if
  • Common uses: And (&&), Or (||), Not (!=)
  • Conditions & Evaluations: < , > , <= , => , = , == , ===, !=
